# Little Joe's Tree Services — Website

A world-class local-service website for **Little Joe's Tree Services**, a Hampshire
tree surgery business. Built with Next.js 15 (App Router) and **exported as a fully
static site** (`output: 'export'`) — TypeScript, Tailwind CSS, and a brand design
system derived from the company logo. `npm run build` produces a static `./out`
folder you can host anywhere; no Node server is required at runtime.

---

## ⚠️ Before this builds: free up disk space

This machine's `C:` drive was **100% full** when the site was scaffolded, so
`npm install` could not complete (`ENOSPC`). All source code is written and ready —
you just need headroom to install dependencies and build.

```powershell
# 1. Free up at least ~2 GB on C: (empty Recycle Bin, clear Downloads, etc.)
#    Quick wins:
#      - cleanmgr  (Disk Cleanup)
#      - npm cache clean --force
#      - clear old node_modules in other projects

# 2. Then do a clean install (the partial node_modules may be incomplete):
Remove-Item -Recurse -Force node_modules, package-lock.json -ErrorAction SilentlyContinue
npm install

# 3. Run it (dev) or build the static site
npm run dev      # http://localhost:3000
npm run build    # outputs ./out  (preview with: npm run preview)
```

**Or skip local builds entirely:** connect the repo to Netlify / Cloudflare Pages /
Vercel and let them build in the cloud — this full-disk machine never has to build.
Build command `npm run build`, publish directory `out`.

## Configuration (`.env.local`)

Copy `.env.example` → `.env.local`. Everything degrades gracefully if unset.

| Variable | Purpose |
| --- | --- |
| `NEXT_PUBLIC_SITE_URL` | Canonical base URL (canonicals, sitemap, OG) |
| `NEXT_PUBLIC_GA4_ID` | GA4 Measurement ID — GA4 only loads if set |
| `NEXT_PUBLIC_GSC_VERIFICATION` | Search Console HTML-tag token (optional) |
| `NEXT_PUBLIC_WEB3FORMS_KEY` | Contact-form delivery via Web3Forms (free, no backend) |

## Project structure

```
site-config.ts          Single source of truth (business data, links, brand)
content/homepage.md      Approved copy (reference)
docs/                    UI direction, SEO brief, agent prompt
lib/                     services, locations, reviews, faqs, schema, seo, analytics
components/              Header, Footer, Hero, sections, ContactForm, ui/ primitives
app/                     Routes (App Router) + sitemap.ts, robots.ts, manifest.ts
  services/[slug]        8 data-driven service pages (pre-rendered)
  areas/[town]           24 data-driven location pages (unique local content)
scripts/process-logo.mjs Logo → transparent PNG + app icons
public/_headers          Security/cache headers for Netlify/Cloudflare hosts
public/brand/            Generated logo assets
out/                     Static build output (generated by `npm run build`)
```

## What's implemented

- **Static export** — every page pre-rendered to HTML; deploy the `out/` folder to
  any static host. No server, no runtime cost.
- **Pages:** Home, About, Contact, Reviews, FAQ, Services (index + 8), Areas (index + 24).
- **SEO:** unique titles/descriptions, self-referencing canonicals, one H1/page,
  `sitemap.xml`, `robots.txt`, JSON-LD (LocalBusiness, WebSite, Service, FAQPage,
  BreadcrumbList), breadcrumbs, internal linking.
- **Performance:** `next/font` (no layout shift), `next/image` (CLS-safe), lazy
  video facade (no iframe until click), restrained motion (respects reduced-motion).
- **Conversion:** clickable `tel:` everywhere, free-quote CTAs, contact form (Zod
  validation + optional photos) delivered via **Web3Forms** (no backend), GA4 events
  (call, form, review, FB, video).
- **Accessibility:** skip link, focus styles, labelled fields, keyboard-friendly nav.
- **Brand:** forest-green + bronze design system, Fraunces/Inter type, layered surfaces.

## ✅ Manual items before go-live

1. **Free disk space + `npm install`** (see top) — or build on the host.
2. Set `NEXT_PUBLIC_SITE_URL` to the real domain; confirm www vs non-www.
3. Add a **Web3Forms key** (`NEXT_PUBLIC_WEB3FORMS_KEY`) — free at web3forms.com,
   set the destination email there — and test the contact form.
4. Add **GA4 Measurement ID** (`NEXT_PUBLIC_GA4_ID`) and verify events fire.
5. **Search Console:** verify the property (tag/DNS/file), submit `/sitemap.xml`.
6. Confirm the **exact business name** styling (see PROJECT-BRIEF).
7. Paste **genuine reviews** into `lib/reviews.ts` (we never invent reviews).
8. Add **real job photos** to service/area pages when available.
9. Add a **full postal address** to the schema if the business has a premises.
10. Replace the footer copyright year if needed (`components/Footer.tsx`).

## Deploy (static)

`npm run build` → upload the **`out/`** folder to any static host:

| Host | How |
| --- | --- |
| **Netlify / Cloudflare Pages** | Connect repo · build `npm run build` · publish `out` · reads `public/_headers` |
| **Vercel** | Detects Next automatically; static export served from the edge |
| **GitHub Pages / S3 / any CDN** | Upload `out/`. For bare hosts, consider `trailingSlash: true` in `next.config.mjs` for cleaner routing |

Set the env vars (`NEXT_PUBLIC_*`) in the host's dashboard before building.
